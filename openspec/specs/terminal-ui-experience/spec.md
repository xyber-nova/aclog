## Purpose

为交互式终端页面提供统一的视觉主题、设计语言、导航约定和工作流键位，使 sync、record、browser 和 stats 等页面共享一致的阅读结构与交互语义。

## Requirements

### Requirement: 交互式终端页面必须共享统一的视觉主题
系统 MUST 为交互式终端页面提供共享的视觉主题，统一定义 verdict、告警、失效状态、强调信息、提示信息和选中态的语义样式，而不是由各页面分别硬编码颜色与高亮行为。

#### Scenario: 不同页面展示相同语义状态
- **WHEN** 用户分别进入 sync、记录选择器、浏览工作台或 stats 界面
- **THEN** 相同语义的状态必须使用一致的视觉表达
- **AND** `AC`、`WA`、告警、失效项和选中项不得在不同页面中呈现互相冲突的颜色语义

### Requirement: 交互式终端页面必须使用统一的训练工作台设计语言
系统 MUST 采用统一的训练工作台设计语言，而不是将各页面实现为彼此独立的工程表格页。该设计语言 MUST 使用冷静专业的文案语气、中等层级的面板感，以及以文字标签为主并辅以少量符号的状态表达。

#### Scenario: 页面展示模式、状态和告警
- **WHEN** 用户在交互式终端页面中查看模式标签、状态标签、空状态或告警信息
- **THEN** 页面必须优先使用可读的文字标签表达这些语义
- **AND** 颜色与少量符号只能作为扫读增强，而不得成为唯一信息载体

### Requirement: 交互式终端页面必须提供统一的导航与帮助约定
系统 MUST 在交互式终端页面中统一支持现有主键位语义，并额外提供 `j/k` 导航别名与 `?` 帮助切换，使用户无需为不同页面重新学习导航方式。

#### Scenario: 用户使用导航别名
- **WHEN** 用户在支持列表移动的终端页面中按下 `j` 或 `k`
- **THEN** 系统必须分别将其视为向下和向上移动
- **AND** 该行为必须与方向键导航等价

#### Scenario: 用户查看帮助与图例
- **WHEN** 用户在任一交互式终端页面中按下 `?`
- **THEN** 系统必须切换当前页面的帮助或图例显示状态
- **AND** 帮助显示不得改变当前页面的业务选择结果

### Requirement: 交互式终端页面必须共享层级一致的工作流键位
系统 MUST 在交互式终端页面中统一采用层级一致的工作流键位：`Enter` 用于进入下一层或确认当前焦点动作，`Esc` 用于返回上一层，`q` 用于直接退出当前工作台，`Tab` 用于同层模式切换；当某个工作台存在两个以上的同层模式时，`Tab` MUST 在全部同层模式之间循环切换；历史返回键如 `b` 可以保留为兼容别名，但不得继续作为主提示键位。

#### Scenario: 用户从子视图返回
- **WHEN** 用户在具有层级结构的交互式终端页面中位于子视图
- **THEN** 按下 `Esc` 必须返回上一层视图
- **AND** 系统不得把 `Esc` 同时解释为其他业务动作

#### Scenario: 用户从根视图退出
- **WHEN** 用户在某个交互式终端工作台的根视图中按下 `q`
- **THEN** 系统必须直接退出该工作台
- **AND** 如果用户在根视图中按下 `Esc`，系统可以将其解释为退出当前工作台

#### Scenario: 多同层模式工作台循环切换
- **WHEN** 用户在存在三个或以上同层模式的终端工作台中按下 `Tab`
- **THEN** 系统必须在这些模式之间按固定顺序循环切换
- **AND** 当前模式必须在页面上下文中被清晰标识

### Requirement: 交互式终端页面必须按一致的阅读结构组织内容
系统 MUST 以一致的阅读结构组织交互式终端页面，至少明确区分上下文信息、主体内容、摘要或详情区域以及操作提示区域，避免用户只能依赖零散正文理解当前状态。

#### Scenario: 页面展示当前对象上下文与可执行动作
- **WHEN** 用户打开任一交互式终端页面
- **THEN** 页面必须先展示当前问题、文件、工作区或视图上下文
- **AND** 页面必须显式展示当前可执行动作或快捷键提示
- **AND** 用户不得只能通过猜测正文来理解页面下一步操作

### Requirement: 全局训练工作台首页必须沿用统一终端工作台设计
系统 MUST 让全局训练工作台首页成为统一 terminal workbench family 的一部分，复用现有的视觉主题、阅读结构、帮助语义和导航键位，而不是为首页实现另一套独立的交互风格。

#### Scenario: 首页展示统一的工作台层级
- **WHEN** 用户打开全局训练工作台首页
- **THEN** 页面必须展示工作区上下文、主入口列表、当前选中入口的摘要或说明，以及操作提示区域
- **AND** 用户不得只能通过猜测正文理解首页下一步可以做什么

#### Scenario: 首页遵循统一导航与帮助键位
- **WHEN** 用户位于全局训练工作台首页并按下 `Enter`、`Esc`、`q`、`?`、`j` 或 `k`
- **THEN** 系统必须继续遵循统一终端工作台的确认、退出、帮助切换和移动语义
- **AND** 该行为必须与其他终端工作台保持一致

#### Scenario: 首页与其他页面共享主题语义
- **WHEN** 全局训练工作台首页展示空状态、强调信息或可恢复 sync 提示
- **THEN** 这些语义必须使用与其他终端工作台一致的文字层级与视觉语义
- **AND** 系统不得为首页单独引入冲突的颜色或提示风格

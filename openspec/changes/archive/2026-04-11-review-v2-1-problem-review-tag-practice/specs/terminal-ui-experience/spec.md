## MODIFIED Requirements

### Requirement: 交互式终端页面必须共享层级一致的工作流键位
系统 MUST 在交互式终端页面中统一采用层级一致的工作流键位：`Enter` 用于进入下一层或确认当前焦点动作，`Esc` 用于返回上一层，`q` 用于直接退出当前工作台，`Tab` 用于同层模式切换；当某个工作台存在两个以上的同层模式时，`Tab` MUST 在全部同层模式之间循环切换；历史返回键如 `b` 可以保留为兼容别名，但不得继续作为主提示键位。

#### Scenario: 用户从子视图返回
- **WHEN** 用户在具有层级结构的交互式终端页面中位于子视图
- **THEN** 按下 `Esc` 必须返回上一层视图
- **AND** 系统不得把 `Esc` 同时解释为其他业务动作

#### Scenario: 用户从根视图退出
- **WHEN** 用户在某个交互式终端工作台的根视图中按下 `q`
- **THEN** 系统必须直接退出该工作台
- **AND** 如果用户在根视图中按下 `Esc`，系统可以将其解释为退出当前工作台

#### Scenario: 多同层模式工作台循环切换
- **WHEN** 用户在存在三个或以上同层模式的终端工作台中按下 `Tab`
- **THEN** 系统必须在这些模式之间按固定顺序循环切换
- **AND** 当前模式必须在页面上下文中被清晰标识
